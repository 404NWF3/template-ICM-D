**一、前置信息说明（全题型适配）**

1.  我要解决一个数学建模问题，需解决以下题目：（参考附件内容）
2.  擅长编程语言/工具：擅长Python语言SciPy/Pandas/NetworkX等库，也会使用COPT求解器。
3.  需求重点：模型创新性、 代码可执行性（含注释与可视化） 、数据处理与图表生成 、论文写作素材（适配学术论文结构） 、模型检验、鲁棒性分析、灵敏度分析。
4.  追求强创新与逻辑闭环，同时也追求解决问题过程中的故事性。

**二、题目分析阶段（全题型通用逻辑）**

1.  逐句解析题目：区分「背景信息」「核心问题」「已知条件（含数据/参数/规则）」「约束条件（含物理规律/场景规则/资源限制）」，用表格分列呈现；重点标注关键逻辑链（如物理过程-参数依赖、决策变量-约束对应、数据字段-分析目标关联），明确核心遵循的原理（物理定律/优化逻辑/统计规律/生态机理等）。

2.  小问划分与目标拆解：列出各小问的「直接目标」「隐含目标」（如表面求数值结果，实际为后续建模铺垫）、「输出要求」（如数值结果/图表/公式/方案表），说明各小问的关联性（如基础模型→场景拓展→多目标优化）。

3.  逻辑关系梳理：用结构化思维导图（文字描述节点与关联）展示参数/变量/目标之间的依赖关系，标注传递路径（如输入参数→中间变量→目标变量、数据预处理→模型构建→结果验证）。

4.  建模方法定位：明确各问题属于「机理分析类」「预测建模类」「优化决策类」「统计分析类」「网络构建类」，结合题目场景说明判断依据，标注是否需要“机理/逻辑定框架+数据提精度”（很多优秀的论文都需二者结合）。

**三、模型选择阶段（基础+创新双方案）**

针对每个小问，推荐2种模型方案（1种基础模型，可以基本地解决问题，容易实现，相对便于理解但是缺乏创行+1种创新模型，有一定的创新型与挑战性），紧扣题目核心逻辑，禁止脱离场景的纯理论模型：

-   方案1（基础适配，保分核心）：经典模型+适配改进点（如「物理机理类：ODE+参数校准；决策优化类：整数规划+约束优先级调整；统计分析类：线性回归+特征筛选；网络类：最短路径+动态权重修正」），确保模型可解释、无逻辑漏洞。

-   方案2（创新融合，冲奖关键）：跨方法融合模型（如「机理模型+机器学习（LSTM/CNN/随机森林）；优化模型+启发式算法（遗传算法/粒子群）；统计模型+集成学习；网络模型+多目标规划」），突出“框架定逻辑、数据提精度、创新提价值”。

1.  每种模型详细说明（适配论文论述）：

-   核心原理：1段通俗语言描述，结合题目场景对应的核心逻辑（物理规律/优化逻辑/统计原理/生态机理），明确模型与场景的适配关系。

-   适配性：为何适合本题，如何匹配数据/参数/约束特征，是否覆盖单一与复杂场景需求。

-   创新点：与传统模型的差异，对解题的实际价值（如动态参数引入、多方法互补、场景化修正），非形式化创新。

-   局限性与规避方案：可能的误差来源（简化假设、数据不足），结合题目特点给出修正方案（敏感性分析、数据校准、约束松弛）。

2.  可视化流程：用文字描述流程图，包含「场景抽象→变量定义→模型构建→数据适配→求解迭代→结果验证」的决策点（如「数据是否满足假设→是：直接建模；否：引入修正项」），标注每个步骤的输出物（公式/参数/图表）。

3.  最后帮我输出一个表格，表格内容为：问题X，问题描述、主要解题思路、方案一模型及原因、方案二推荐模型及原因

**四、数据处理阶段**

若题目提供数据可进行该步骤，若未提供数据请在全网自主寻找合适数据并对数据进行预处理操作等，但需要注明数据来源！

1.  题目数据说明：【列出数据（含单位/维度/类型）或标注附件，明确数据类型：如物理参数数据、时序数据、离散决策数据、网络节点数据、环境监测数据、结构化/半结构化数据】。

2.  预处理方案（结合数据特性，避免脱离场景）：

-   缺失值处理：根据数据类型选择方法（均值/中位数/众数填充、线性/多项式插值、KNN填充），说明依据（如时序数据用线性插值、物理参数用均值填充），给出处理后数据（标注单位）。

-   异常值处理：用Z-score法/IQR法/箱线图检测，结合场景逻辑说明保留/修正/删除的理由（如违背物理守恒/生态阈值的异常值删除），附处理结果描述。

-   转换与特征处理：说明标准化/归一化/编码（One-Hot/Label Encoding）的必要性，给出转换公式；若为图像/网络/空间数据，说明特征提取方法（边缘检测、节点-边构建、空间插值），确保特征贴合场景意义。

-   数据补充：区分「必须补充」（核心参数/标准常数/行业规范）和「可选补充」（辅助数据），推荐可落地获取途径（公开数据集Kaggle/UCI、行业报告、权威机构数据EPA/UNEP）；无法获取时，基于场景逻辑合理假设补充（详细标注依据）。

-   生成属于预处理完整的源代码：代码中必须包含清晰的本地数据上传路径提示和清晰的处理完成的数据输出路径提示，使用醒目标注（如注释、变量名）指引学生替换为自己的本地数据路径；代码包含详细注释，标注每个步骤的目的、方法和参数含义；代码中最好包括数据可视化模块，包括数据预处理前的可视化分析和数据预处理后的可视化分析等，可视化分析要看是否有必须做，如有必要则在代码中包括，可视化分析图片质量要求比较高，美观性较强！

3.  将预处理完数据保存为通用格式（CSV优先）直接展示处理后数据的前10行和后5行；

-   提供多种虚拟下载链接（注明文件名称、大小、格式）；
-   提示我如何验证处理后数据的完整性。
-   总结整个数据预处理过程，形成数据预处理结论

**五、模型建立阶段（逻辑-公式衔接）**

**结合前面针对每个小问推荐的两种方案，最终选择每小问【方案X】；**

结合模型与处理后的数据，参考以下流程，详细说明每小问完整的建模步骤，突出场景逻辑与数学公式的强关联，分别按照每小问顺序详细阐述！

1.  变量定义：用表格明确「决策变量/中间变量/目标变量/参数」，标注类型（连续/离散/0-1）、单位、约束范围，说明对应的场景意义（如「c(x,y,t)：污染物浓度，mg/m³，约束≤生态阈值」），区分“已知参数/校准参数/假设参数”。

2.  假设条件：3-5个核心假设，每个假设含“内容+依据+对模型的影响”（如「假设短期内气象稳定，依据：题目聚焦核心机理，影响：简化计算，通过鲁棒性分析覆盖波动」），避免无依据假设。

3.  公式推导：分步骤推导核心公式（目标函数/约束方程/机理方程），每步标注逻辑依据（物理规律/优化目标/统计原理），公式标注规范（符号/下标统一）。

4.  建模流程图：用文字描述「场景抽象→变量定义→公式推导→约束设定→目标确立→参数校准」的逻辑链，标注关键节点（如「验证公式与核心逻辑一致性」），便于转化为论文图表。

**六、模型求解阶段（代码落地+可视化）**

1.  源代码编写：优先选择易调试、可复现的方法，按编程语言规范编写。

2.  求解步骤：按「数据输入→参数初始化（标注意义/范围/校准方法）→模型调用→结果输出」说明，标注注意事项（如参数约束范围、求解器参数设置、步长选择）。

3.  代码实现：

-   零编程基础：提供可直接运行的完整代码（标注依赖库及安装命令），含详细注释（每段对应场景逻辑）、异常处理、可视化模块，关键行说明“为何这么写”及场景意义。

-   使用ipynb文件，按功能添加python与markdown代码块，代码编写不要采取先按功能定义所有python函数之后在“主执行模块”依次执行所有函数的模式

4.  可视化分析（适配论文图表要求）：根据场景选择图表（折线图/柱状图/拟合图/网络拓扑图/空间分布图/热力图），图表含规范标题、坐标轴标签（带单位）、图例、误差线，图下标注关键结论（如「使用频率增加10%，磨损深度增加0.2mm，符合物理规律」），满足专业论文的图表审美标准。

5.  按照上述建模步骤步骤，结合已上传的真实数据集运行代码得到相关结果，结果需要与数学建模问题的背景与需求高度对应，能够完成所有要求，并对各小问代码运行结果结合题目要求进行初步的解读和总结。

**七、结果分析阶段（场景意义+论文论证）**

针对求解结果，给出「基础分析」和「深层分析」，避免纯数值解读：

1.  基础分析：

-   数值解读：说明结果的场景意义、实际价值（如「最优方案成本降低15%，满足所有约束」），而非罗列数值。

-   统计/指标描述：计算均值/方差/拟合优度（R²）/MSE/AUC/约束满足度等，说明结果分布与场景的匹配性。

2.  深层分析（冲奖核心）：

-   关联性：结果与场景约束、核心逻辑的匹配度（如「满足物理守恒/生态阈值/决策目标，无逻辑矛盾」），验证模型合理性。

-   敏感性/特征重要性分析（必做）：分析2-3个核心参数/特征对结果的影响（如「运输成本增加10%，最优路径调整」），附图表与结论。

-   实际意义与延伸：结合题目背景提炼结论，关联实际应用场景（如「为资源调度/污染防控/政策制定提供数据支撑」），提升论文深度。

**八、模型检验与改进阶段（强化评分点）**

1.  有效性检验：

-   方法：选择适配模型的检验指标（拟合优度/约束满足度/模拟精度/多场景适配性），结合交叉验证/实地数据验证，避免单一指标。

-   代码与结果：提供检验代码（如5折交叉验证、残差分析、多场景代入），输出结果解读（如「平均MSE=0.02，约束满足度100%，模型有效」）。

2.  改进方向（体现思维完整性）：

-   模型短板：具体改进点（引入动态因子、优化算法、增加模块），而非泛泛而谈。

-   假设不合理：修正方案（放松假设、引入波动因子、数据校准）。

3.  鲁棒性分析与灵敏度分析（必做）：改变核心参数（波动±10%）、添加数据噪声、模拟极端场景，观察结果变化幅度，说明模型抗干扰能力，结合场景说明实际意义（如「参数波动10%，结果偏差≤5%，适配实际误差」），附图表。

4.  优缺点评价：补充模型优缺点评价，系统客观评价模型优劣：

-   模型优点（3-5条）：从创新、适配、可靠性维度，结合AI求解数据阐述（如“融合Bayesian反演与PSO算法，参数校准精度提升25%；适配零值膨胀数据，预测覆盖度达88.6%，优于传统机器学习模型”）每条均有数据支撑。

-   模型缺点（2-3条）：客观说明局限（如“未覆盖极端场景下的材质老化影响；部分参数依赖经验值，样本量不足时精度略有下降”），不泛泛而谈。
